# 0003. 3 層ソース構成と日経除外

- 状態: 採用
- 日付: 2026-04-24
- 関連 PR: #1(Sprint 2 設計書)

## 背景

EduWatch JP の MVP では教育ニュースを自動集約するが、ソースの取捨を最初に決めないと、(a) 不要な情報で読者の意思決定を妨げる、(b) クロール負荷が肥大化する、(c) 編集者の週次ダイジェスト作業が破綻する、という問題が起きる。

候補ソースは大きく以下の 3 系統あった:

1. 一次情報(政府 / 国研 / 国際機関): 文科省 / 中教審 / NIER / OECD / EEF
2. 主要教育メディア: 朝日 EduA / 毎日 教育 / 読売 こどもと教育 / 共同通信 教育 / 日経 教育
3. 話題性シグナル: Yahoo ニューストピックス / Google Trends / X(旧 Twitter)

それぞれの位置づけと取得方式を決める必要があった。

## 検討した選択肢

- **A) 全候補を同じ重みで取得、編集側で取捨**
- **B) 3 層構成にして層別に役割を分け、有料壁ソースは除外**
- **C) 一次情報のみに絞り、メディア・話題性は将来検討**

## 決定

**B) 3 層構成 + 日経除外** を採用。

### 第 1 層 — 一次情報(自動収集)

- 文科省(`mext.go.jp/b_menu/news/index.rdf`)
- 中教審(文科省 RSS から「中教審 / 答申」で派生抽出)
- NIER(`nier.go.jp/02_news/rss.xml`)
- OECD(後述、現状除外)
- EEF(後述、現状除外)

### 第 2 層 — 主要教育メディア(自動収集、HTML スクレイピング)

- 朝日 EduA / 毎日 教育 / 読売 こどもと教育 / 共同通信 教育

### 第 3 層 — 話題性シグナル(自動収集しない、編集者の参考)

- Yahoo ニューストピックス
- Google Trends
- 取得は編集者が週次ダイジェスト作成時に手動で参照

### 日経除外の理由

日経 教育は **有料壁** のため、自動収集してもタイトルとリードしか取得できず、二次的なリンク提供が読者の利便性に貢献しない(クリックしても課金壁に阻まれる)。本サイトの編集方針(誰でも検証可能な一次情報を提示する)とも整合しない。

### コミットボット名

`edu-watch-bot <notify@edu-evidence.org>` で確定(`@` の重複なく自然、Email Routing で運営者へ転送)。

## 帰結

### 良い帰結

- 一次情報を最上位に置くことで、誤情報の伝播リスクを最小化
- 第 3 層を「自動収集しない」と切り分けたことで、Yahoo / Google の利用規約への配慮も明確
- 日経除外で、課金壁に阻まれる読者ストレスを避けられる
- 設計書 `docs/sprint-2-design.md` §3 で全 9 ソースの URL / 方式 / 頻度が一覧化されている

### トレードオフ

- 主要メディア 4 紙を HTML スクレイピングで取得するため、サイト構造変更時の保守が必要
- 日経除外により、ビジネス層に届く教育記事の捕捉率がやや下がる
- 第 3 層を自動化しないため、編集者の工数が週次ダイジェスト作成時に集中する

## 撤回 / 再検討の条件

- 日経が無料公開の教育セクションを開設した場合
- 主要メディアが RSS を再開した場合(HTML スクレイピングを RSS に置き換える)
- Yahoo / Google が公式 API を提供した場合(第 3 層を自動化に格上げ検討)
