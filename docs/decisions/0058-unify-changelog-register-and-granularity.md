# 0058. changelog の文体(敬体)と粒度をファミリー統一する(edu-law ADR 0022 ミラー)

- 状態: 採用
- 日付: 2026-06-22
- 関連 PR: docs/changelog-register-adr
- 関連 ADR: edu-law 0022(原本・本 ADR のミラー元) / edu-law 0017(更新情報の正本を changelog データに一元化)

## 背景

edu-law では ADR 0017 で changelog データをサイト更新情報の正本とし、`/changelog/`・`/rss.xml`・トップ「最近の更新」を導出している。3 サイトとも同様に changelog を持つが、ファミリー横断で書き方が 2 軸で不統一だった。

文体(register):

- edu-evidence(葉): 敬体(「〜しました」)、約 130 件
- edu-watch(双葉): 体言止め中心(「〜整理」)、40 件
- edu-law(根): 常体(「〜した」)、26 件

粒度(granularity): 平均字数では edu-law が最も長く edu-watch が最も短いが、突出した差はない。ただし edu-evidence の一部(Technical Appendix 追加バッチ)は、効果量・メタ分析名・引用などページ本文に載る詳細を changelog に再掲しており、1 件 130 字を超えるなど冗長だった。

changelog は RSS でも配信され、姉妹サイトを横断して読まれうる(4 サイト横断リピート訪問性分析 2026-06、edu-law ADR 0017 背景)。文体・粒度が揃っていないと横断時の体験が不揃いになる。

## 決定

ファミリーの changelog エントリ本文を、次の 2 軸で統一する。

1. **文体は敬体(です・ます調)。** 基準は既に敬体で運用されている edu-evidence に合わせる(「edu-watch の UI/UX は edu-evidence に揃える」方針と同様、葉=基準)。文末述語を敬体にし(「〜した」→「〜しました」、「〜できる」→「〜できます」、体言止めは連用中止で接続して文末を敬体にする)、種別ラベル・リンク・日付などの構造は変更しない。
2. **粒度は簡潔に。** 1 エントリは「何が変わったか＋簡潔な理由＋(対象ページが一意なら)該当ページへのリンク」に収め、ページ本文に載る詳細(効果量・研究名・引用)を changelog に再掲しない。詳細はリンク先のページに委ねる。

本 ADR は edu-law ADR 0022 のミラーであり、3 リポ共通の規約として記録する。

## なぜこの判断にしたか

- 敬体は対象読者(小学校教員)に最も穏当で読み手に優しい。
- 最大資産 edu-evidence(約 130 件)が既に敬体のため、文体の基準をそこに置けば書き換えは edu-watch 40 + edu-law 26 に最小化できる。
- ファミリーのスタイル統一は既存運用(aria/data 属性・アクセントバーを 3 リポで ADR 統一)と一貫する。
- 粒度を分けて規定するのは、文体(敬体)と詳細さ(再掲の有無)が別問題であり、「evi を基準にする = evi の冗長さも基準にする」ではないことを明示するため。

## 帰結

- edu-law は既存 26 件を敬体へ変換済み(register のみ、字数は増やさない。原本 ADR 0022)。
- **edu-watch(本リポ)は体言止め中心の 40 件を PR #312 で敬体へ変換済み。**
- edu-evidence は文体の基準で原則そのまま(敬体でない混在があれば是正)。粒度規約に反する冗長な既存エントリ(Technical Appendix バッチ)の圧縮は、情報欠落・破損リスクを避けるため遡及一括では行わず、必要なら任意の別タスクとする。
- 今後の新規エントリは敬体・簡潔粒度で書く(「PR ごとに changelog 更新」運用に文体・粒度の規約を加える)。

## トレードオフ / 既知のリスク

- 敬体はやや冗長で、edu-watch の見出し的な簡潔さは一部失われる。統一性と穏当さを優先する。
- 既存ユーザー向け本文の一括編集は破損リスクがある(DELEGATE-52)。リポ別・サンプル承認・build 確認で慎重に進める。

## 撤回 / 再検討の条件

- changelog を機械生成(コミットメッセージ等からの自動導出)へ切り替える場合は、生成側の文体・粒度規約として再定義する。
