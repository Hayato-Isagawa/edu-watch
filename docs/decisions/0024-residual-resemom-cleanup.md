# 0024. ADR 0019 取りこぼしの予防的封じ込め

- 状態: 採用
- 日付: 2026-05-02
- 関連 ADR: 0018(対象読者を学校教員・教育関係者に統一)、0019(対象読者の遡及適用とリセマム NG ワード再設計)、0020(削除した記事 ID の永続追跡 + 再取り込み防止)
- 関連 PR: TBD(本 ADR の実装)

## 背景

ADR 0019(2026-05-01、PR #67)で resemom 38 件を遡及削除し、`isExcludedByTitle()` の NG_PATTERNS を 14 → 26 に拡張した。検証時点で「34 件中 27 件(79%)を新パターンで自動捕捉、残り 7 件は汎用イベント語彙で正規化困難 → editor review に委ねる」と記録した。

ADR 0020(2026-05-02、PR #71)では cron で復活した 6 件を永続 denylist + narrow パターン 5 件追加で塞いだ。

2026-05-02 セッション後半、「残り 1 件あるはず」とのユーザー指摘から PR #67 削除 38 件全てに現行 NG_PATTERNS を適用し直したところ、**実際には 7 件取りこぼしのまま**(うち 2 件は ADR 0020 denylist で二重防御済み、残り **5 件は無防備**)であることが判明:

| # | タイトル | denylist | 防御状態 |
|---|---|---|---|
| ✅1 | ちばみんフェス | あり | 二重防御 |
| ✅2 | 大学生への仕送り | あり | 二重防御 |
| 🚨3 | 立命館大「衣笠アートヴィレッジ フェス」 | なし | **無防備** |
| 🚨4 | スポーツ通訳士の仕事を紹介…元プロ野球選手ら登壇 | なし | **無防備** |
| 🚨5 | GW 明けは子供のやる気低下を実感 | なし | **無防備** |
| 🚨6 | JAXA「油井宇宙飛行士ミッション報告会」 | なし | **無防備** |
| ⚠️7 | 国際天文学オリンピック2026、灘高ら日本代表 5 人決定 | なし | 判断保留 |

無防備な 4 件は現状 RSS で再配信されていないが、開催日(5/19、5/24、5/31 など)が近づいて RSS 側で再露出されると、現行 NG_PATTERNS と denylist のいずれにも引っかからずに main 復活する可能性がある。

「国際天文学オリンピック2026 日本代表 5 人決定」については `scripts/check-resemom-filter.ts` で `expected: false`(残す)とテストケース化されており、ADR 0019 PR #67 での実削除と判断が **矛盾している**。本セッションで再判断した結果、**STEM 進路情報は学校教員の進路指導の参考になる** ため `check-resemom-filter.ts` の判断(残す)を正とし、ADR 0019 の削除は誤りだった位置付けに統一する。

## 検討した選択肢

### A. 4 件を denylist + NG_PATTERNS narrow パターン拡張(採用)

メリット:
- 復活防止の二重防御(parser 層 + denylist 層)
- 同種記事の将来流入を NG_PATTERNS で抑止
- ADR 0019 §「7 件は汎用語彙で困難」の判断を、本 ADR で「ADR 0024 で 4 件分を narrow パターン化、残り 3 件は二重防御済み or 判断保留」に更新

デメリット / 緩和策:
- denylist が 6 → 10 件に増えるが、JSON 読み込みコストは無視可能
- narrow パターン追加で false positive のリスク → `check-resemom-filter.ts` に「弾いてはいけない」ケース 2 件(汎用「報告会」「登壇」)を追加して回帰防止

### B. 7 件全て denylist 化、`check-resemom-filter.ts` の expected も書き換え(却下)

却下理由:
- 「国際天文学オリンピック 2026 日本代表決定」は STEM 進路情報として教員視点で関連する余地があり、ADR 0019 §「残す 13 件」の境界線判断と整合させるべき
- `check-resemom-filter.ts` の「弾いてはいけない」ケース(灘高ら日本代表決定)を残すことで、将来同種の STEM オリンピック記事も自動的に保護できる

### C. denylist のみ追加、NG_PATTERNS は変更しない(却下)

却下理由:
- 同種記事(「立教大 ◯◯フェス」「東大 ◯◯ミッション報告会」など)が将来流入した場合、毎回 editor review で削除する運用負担が続く
- narrow パターン化することで自動防御を継続的に効かせられる

### D. ADR 0019 §「残り 7 件」を放置(却下)

却下理由:
- ADR 0020 で「6 件中 4 件は narrow パターンで自動捕捉できた」と判明した実績から、残り 5 件も狭い表現で対応可能(本セッションで実証)
- 実際に開催日近接で RSS 再露出される蓋然性は低くないため、予防的に塞ぐ価値は明確

## 決定

A を採用。

### (1) `src/data/excluded-article-ids.json` に 4 件追加

| id | 削除根拠(ADR 0019 §) |
|---|---|
| `resemom-2026-04-22-3f4390994c986325` | A 親子・体験イベント(立命館大「衣笠アートヴィレッジ フェス」5/31) |
| `resemom-2026-04-23-542a56a843bc4a88` | A 親子・体験イベント(JAXA 油井宇宙飛行士ミッション報告会 5/19) |
| `resemom-2026-04-23-cabd0fb9ea285373` | A 親子・体験イベント(スポーツ通訳士登壇、元プロ野球選手 5/24) |
| `resemom-2026-04-24-f1b89b7a74bf3d6d` | E 家庭ライフスタイル(GW 明け子供のやる気低下) |

denylist のスキーマ / loader / fetch-news 統合は ADR 0020 のものをそのまま流用。`reasons` には ADR 0024(本 ADR)を明記して監査可能性を保つ。

### (2) `src/lib/sources/resemom.ts` NG_PATTERNS に 4 個追加

| パターン | 捕捉対象 |
|---|---|
| `/アートヴィレッジ/` | 立命館大「衣笠アートヴィレッジ フェス」型 |
| `/(?:JAXA\|宇宙飛行士).{0,15}報告会/` | JAXA / 宇宙飛行士 + 報告会(汎用「報告会」は弾かない) |
| `/(?:プロ(?:野球\|サッカー\|バスケ).{0,5}選手\|タレント).{0,5}登壇/` | 元プロ選手 / タレント登壇イベント(汎用「登壇」は弾かない) |
| `/(?:GW\|夏休み\|冬休み\|春休み)明け.{0,15}子(?:供\|ども)/` | 季節区切り + 子供主体の家庭ライフスタイル |

特に「報告会」「登壇」のような汎用語彙には narrow 化が必須(教員向け学術報告会・教育シンポ登壇者などを誤って弾かないため)。

### (3) `scripts/check-resemom-filter.ts` 拡張

- 取りこぼし 4 件を `expected: true` ケースとして追加
- 汎用「報告会」「登壇」を `expected: false` ケースとして追加(narrow パターンが過剰に効かないことの回帰防止)
- ケース合計 25 → 31 ケース

### (4) 「国際天文学オリンピック」の判断統一

`scripts/check-resemom-filter.ts` の既存ケース「国際天文学オリンピック 2026 日本代表 5 人決定」は **`expected: false`(残す)を維持**。STEM 進路情報として教員視点で関連する余地があるため、ADR 0019 PR #67 の削除判断ではなく本テストケースの判断を正とする。**denylist には追加しない**(再配信されたら自然に main に戻ることを許容)。

### (5) changelog

ユーザー価値が薄い(取りこぼし防止の運用改善)ため `changelog.astro` には **追加しない**。メモリルール 8b「changelog はユーザー価値のある変更だけを書く」に従う。

## 影響と運用

### 効果

- 4 件の RSS 再配信時の復活を確実に防げる(parser 層で弾かれ、もし弾かれなくても denylist で最終ガード)
- 同種記事(立教大 ◯◯フェス / 東大 ◯◯ミッション報告会 / GW 明け子どもケア調査など)の将来流入も自動抑止
- ADR 0019 §「残り 7 件」の運用上の不確実性を解消(2 件 ADR 0020 + 4 件 ADR 0024 + 1 件は判断保留で残す方針確定)

### 監視

- `npm run check:filter:resemom` 31 ケース pass で回帰防止
- `npm run check:excluded-ids` で denylist と article JSON の整合確認
- 火曜 health-check workflow で週次定点監視

## 撤回 / 再検討の条件

- 「国際天文学オリンピック」と類似の STEM 進路情報が今後 article JSON に流入し、サイト一覧で目立った場合 → 「残す境界線」の見直しを別 ADR で検討
- narrow パターンが false positive を生み出す事例が観測された場合 → `check-resemom-filter.ts` のケース追加で回帰防止しつつパターン調整
- `excluded-article-ids.json` が 50 件超に膨張 → 形式や運用フローを再検討、D1 移行と併せて検討

## 参考

- ADR 0019 §「検証結果」で「7 件は汎用イベント語彙で正規化困難」と記録された 7 件のうち、ADR 0020 で 2 件 + ADR 0024(本 ADR)で 4 件 + 判断保留 1 件 = **7 件全て扱いが確定**
- 「国際天文学オリンピック」のテストケース判断と PR #67 の実削除が矛盾していた問題は、本 ADR で「テストケースを正とする」方針で統一
