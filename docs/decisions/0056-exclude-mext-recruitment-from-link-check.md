# 0056. リンクチェックから MEXT 採用公告（非常勤職員）を除外する

- 状態: 採用
- 日付: 2026-06-16
- 関連 issue: #283
- 関連 PR: `fix/lychee-exclude-mext-recruitment`

## 背景

週次の自動リンクチェック（lychee、`.github/workflows/link-check.yml`、毎週月曜 00:00 UTC）は、リンク切れ検出時に `needs-human-review` ラベル付き issue を自動生成する。2026-06-15 の run が生成した #283 は「リンク切れ 4 件」を報告した。

- 報告された 4 entries は実体 1 URL（`https://www.mext.go.jp/b_menu/saiyou/hijyoukin/1423086_00016.html`）。dist 上の 4 箇所（`archive/2026-05-22`・`categories/policy`・`categories/teachers`・`sources/mext`）は同一記事の展開先で、3 件は lychee キャッシュ由来。
- 当該 URL は記事「文部科学省初等中等教育局非常勤職員（時間雇用職員）採用のお知らせ（令和8年7月1日採用）」（`src/data/articles/2026-05-21.json`、`id: mext-2026-05-21-8e058d62175dcc8a`）の `sourceUrl`。
- 2026-06-16 に実測したところ、当該 URL は GET で恒久 404、同じ MEXT ドメインの対照 URL（審議会ページ）は GET 200 を返した。すなわち bot ブロックではなく当該ページが真に削除されている。
- これは偶発的なリンク切れではなく、応募期限付き採用公告という性質上、締切超過後に MEXT がページを削除する設計上必然の 404 である。MEXT 採用公告（`/b_menu/saiyou/`）は揮発性が高く、収集時点では layer 1 の一次情報として正当だが、数週間で 404 化する。
- 記事自体は「2026-05-21 に MEXT がこの採用公告を出した」という史実として正確であり、ニュースアーカイブの記録として保持が妥当。元 URL も当時の一次情報として残す。

kknews（ADR 0055）が「リンクは生きているが CI から到達できない偽陽性」だったのに対し、本件は「リンクが真に消えたが記事は史実として保持したい」ケースで、原因は異なる。ただし「リンクチェッカーが恒常的にノイズを生む既知の一次情報源クラスを除外する」という対処は同型である。

## 検討した選択肢

- **A) 毎回手動でクローズし続ける**（現状）。採用公告は今後も継続的に収集・404 化するため、同種 issue が再発し本物のリンク切れを埋もれさせるノイズになる。
- **B) `sourceUrl` を MEXT 採用一覧等へ差し替える**。個別公告 → 一覧ページで粒度が落ち、記事タイトルと URL の対応が崩れる（sourceUrl は記事そのものを指す一次源原則に反する）。かつ差し替え先 URL の生存も保証されない。
- **C) 当該記事を削除する**。アーカイブの史実性を損なう。採用公告も収集時点の一次情報であり、恣意的な削除は収集方針と一貫しない。
- **D) lychee から MEXT 非常勤採用公告パス（`/b_menu/saiyou/hijyoukin/`）を除外する**（`--exclude`）。kknews（ADR 0055）・edu-evidence で確立済みのパターン。記事と元 URL は保持しつつ、揮発性クラスの偽ノイズだけを止める。

## 決定

D を採用する。`link-check.yml` の lychee args に `--exclude 'mext\.go\.jp/b_menu/saiyou/hijyoukin/'` を追加する。

除外スコープは観測された揮発性クラス（非常勤職員採用 = `hijyoukin`）に限定し、MEXT ドメイン全体や `/b_menu/saiyou/` 全体には広げない。常勤採用など他パスで同種の 404 が再発した場合に、その都度スコープを評価して追記する。

## 影響

- CI から `/b_menu/saiyou/hijyoukin/` 配下の「本物のリンク切れ」を自動検知できなくなる。ただし当該パスは採用公告のみで、いずれ確実に 404 化する揮発性コンテンツのため、検知能力の実質的損失はない。
- MEXT の他パス（審議会・通知・統計等）および他ドメインのリンクチェックは従来どおり機能し、実リンク切れは引き続き検知される。
- 記事 `mext-2026-05-21-8e058d62175dcc8a` とその `sourceUrl` は史実として保持する。
- 将来、常勤採用（`/b_menu/saiyou/` 配下の別パス）等で同種の 404 が出た場合は、本除外のスコープを再評価して追記する。
