# 0018. 対象読者を学校教員・教育関係者に統一する

- 状態: 採用
- 日付: 2026-05-01
- 関連 ADR: 0007(第 2 層を教育専門紙へ転換)を上書き(読者像の再定義)
- 関連 PR: TBD(本 ADR の実装)

## 背景

PRD v1.0(2026-04-24 承認)§3 では対象読者を「小・中学校の現場教員」と「保護者」の 2 層と定義し、サイト各所(`README.md` / `CLAUDE.md` / `src/layouts/Layout.astro` フッター / `src/pages/about.astro`)もこの 2 層を前提に記述してきた。Sprint 2〜4 の運用で、この 2 層構成が実装・編集の両面で齟齬を生じさせていることが確認された。

問題:

- **編集軸が定まらない** — 政策・教員養成・働き方・授業実践は教員に刺さるが、受験・習い事・教育費・家庭学習は保護者に刺さる。両方を同じ日次フィードに混ぜると、ニュース選定基準が「どちらの読者に向けるか」で毎回判断が分かれ、収集ロジック(`isExcludedByTitle()` の NG ワード設計など)も方針が揺らぐ。
- **競合との関係** — 保護者向け教育情報領域は既にリセマムが強く、無料の一次情報ハブとして差別化する余地が小さい。一方、教員向け無料一次情報ハブはニッチが空いている(教育新聞・日本教育新聞は有料壁、文科省公式は導線が悪い)。
- **EduEvidence JP との一貫性** — 姉妹サイト EduEvidence JP は教員専用(指導法エビデンス)。同じ読者層に向けて「ストック(EduEvidence)+フロー(EduWatch)」と時間軸で住み分ける方が、ブランド・SaaS ファネル(運営者の将来計画)としても整合する。
- **ADR 0007 との整合** — 第 2 層を大手紙から教育専門紙(教育新聞・日本教育新聞・教育家庭新聞・リセマム)へ転換した時点で、運用実態は既に教員寄りに傾いている。ADR 0007 §「読者像」自体は教員と保護者を併記していたが、その後の運用判断はすべて教員視点を優先してきた。

## 検討した選択肢

### A. 対象読者を「学校教員・教育関係者」に統一(採用)

教員・指導主事・管理職・教育研究者・教育系記者を含む「学校教育に職業的に関わる人」を一次読者とし、保護者は明示的な対象から外す。保護者向けの一次情報(教育費調査・SNS リテラシー・家庭との連携など)は、**教員視点で関連する場合のみ採用** する基準に切り替える。

メリット:
- 編集軸が単線化し、ニュース選定が機械的に決まる
- リセマム等の既存メディアと正面衝突せず、無料一次情報ハブとして差別化
- EduEvidence と読者層が完全一致し、SaaS ファネル(将来構想)が一直線
- ADR 0007 で進めた教育専門紙シフトと完全整合

デメリット / 緩和策:
- 保護者層の流入機会を失う → 教員視点で関連する保護者向け一次情報は引き続き採用するため、家庭との連携・教育費・リテラシー指導などのテーマは残る
- リセマム由来の保護者向け純情報の取り扱いが厳格化 → 別 PR で `src/lib/sources/resemom.ts` の `isExcludedByTitle()` を見直す。本 ADR では方針確定のみ

### B. 2 層併記を維持(却下)

却下理由: 上記「背景」の通り、編集軸の二重化が運用負荷とブランド曖昧化の原因になっている。Sprint 4 まで併記を続けた結果、リセマム由来の保護者寄り記事を毎回編集判断で篩い分ける負荷が常態化した。

### C. 完全な保護者向け転換(却下)

却下理由: 既にリセマムが強い領域。差別化困難で、運営者の SaaS ファネル(教員業務支援)とも整合しない。

## 決定

A を採用。

### 対象読者の定義(改定版)

**プライマリ読者**: 小・中・高等学校の現場教員(20〜50 代)

**セカンダリ読者**:
- 管理職(校長・教頭)、指導主事
- 教育研究者・大学院生
- 教育系ライター・記者
- 教育行政・教育委員会関係者

**明示的に外す読者層**: 保護者(子育て情報・受験対策・習い事を主目的とする層)。ただし保護者向けの一次情報(調査・政策)で、**教員視点で関連する場合**(家庭との連携・情報モラル指導・進路指導・経済格差と学習機会など)は引き続き採用する。

### 編集ポリシーへの反映

PRD §6.3「掲載しない基準」に以下を追加:

- 保護者の私的関心に閉じる純情報(中学受験対策・幼児教育のおすすめ・習い事ランキング・ファッション/ライフスタイル系の家庭情報)は採用しない。教員視点で関連する場合のみ採用する。

### 文書・UI への反映(本 PR で実施)

- `CLAUDE.md` リード文
- `README.md` リード文
- `docs/PRD.md` §1.3 / §2.1 / §3.1 / §3.2 / §5.1.第 3 層 / §6
- `src/layouts/Layout.astro` フッター説明
- `src/pages/about.astro` hero `description` / 役割節 / 編集ポリシー節
- `src/pages/changelog.astro` に 1 行追加(対象読者を教員に明確化)
- `src/lib/sources/nier.ts` / `resemom.ts` のコメント表現

### 別 PR に分離する変更

- `src/lib/sources/resemom.ts` の `isExcludedByTitle()` フィルタ強化(NG ワード再設計)
- ReseEd(リセマム教員向け姉妹サイト)追加採用判断の優先度引き上げ
- 既存記事 JSON に対する遡及的なフィルタ適用 — 行わない(過去ログは不変)

## 影響と運用

- Sprint 4 の週次ダイジェスト編集ワークフロー(ADR 0017、`digest-fact-checker`)の判断基準が一線化する
- 今後の RSS ソース追加判断は「教員視点で価値があるか」を一次基準とする
- 既に蓄積された記事(`src/data/articles/*.json`)は遡及修正しない。本 ADR 採用以降の収集分から新基準を適用する

## 再検討の条件

以下のいずれかが起きた場合、本 ADR の見直しを検討する:

- 教員専用ニュース層の流入が想定を下回り、保護者層への再アプローチが必要になった場合
- リセマム / ReseEd の運営方針変更により、保護者向け / 教員向けの境界が変化した場合
- EduEvidence / EduWatch / SaaS のブランド境界に関する運営者の事業計画が変更された場合
