# 0040. AI 補助 PDF 要約と編集者最終監修の運用

- 状態: 採用
- 日付: 2026-05-12
- 関連 ADR: 0007(三層ソース運用方針)/ 0008(引用範囲遵守ポリシー、§3 を本 ADR で改訂)/ 0018(対象読者は学校教員・教育関係者)/ 0036(tier 1 ソース基準拡張、公的 PDF 採用根拠)
- 関連 PR: TBD(本 ADR 起票 PR)
- 撤回 / 再検討トリガー: 本 ADR §「撤回 / 再検討の条件」参照

## 背景

ADR 0036 で tier 1 公的一次ソース(政府・自治体・公的研究機関の公表 PDF)を edu-watch の運用範囲に含めた結果、中央教育審議会答申のような 70p+ の長大 PDF を週次 digest で扱う場面が増えた。編集者が原文を全文読了して論点抽出する負担を軽減するため、ローカル Ollama によるダイジェスト化補助を 2026-05-09 〜 2026-05-12 のセッション 11-16 で PoC として検証した。

### 対象 PDF

中央教育審議会 第 251 号答申本体「『令和の日本型学校教育』を担う質の高い教師の確保のための環境整備に関する総合的な方策について」(73p / 91,987 chars)

### chunk 設計と pre-trim

| chunk | pages | chars |
|---|---|---|
| 1 | 1-15 | 16,371 |
| 2 | 16-30 | 18,417 |
| 3 | 31-45 | 16,802 |
| 4 | 46-60 | 15,720 |
| 5 | 61-67 | 10,234 |
| (pre-trim) | 68-73 | — |

末尾 6p(委員名簿等)は pdf-parse 抽出時にグリフ ID 直マッピングの文字化けが集中し、context 末端でモデルが幻覚を呼び込む傾向が観測された(セッション 12 で「峰岡俊明教授」「3,000名」など実在しない固有名詞を捏造)。chunk 設計段階で恒常的に pre-trim する方針を採用。

### モデル選定

| 観点 | qwen2.5:7b(初期採用) | gemma3:12b(再検証採用) |
|---|---|---|
| context | 32K | 128K |
| サイズ | 4.7GB | 8.1GB |
| M1 16GB ロード | 余裕 | ほぼフルロード |
| 平均 tokens/sec | 7.40 | 4.68 |

セッション 14 の qwen2.5:7b fact-check で「教職調整額 10% 以上」(本答申最大の政策的争点、p.50)を含む CRITICAL 欠落 3 件 + HIGH 欠落 1 件を発見。中国系モデルの言語バイアス疑いを排除する目的で gemma3:12b に交換し、5 chunk フル map-reduce を再実行。

### fact-check 結果サマリ(両モデル比較)

| 観点 | qwen2.5:7b(summary-v2) | gemma3:12b(summary-v3) |
|---|---|---|
| 教職調整額 10% 以上 | **CRITICAL 欠落** | ✓ ACCURATE(4 か所言及、p.53 出典) |
| 月45h / 年360h | ✓ ACCURATE | ✓ ACCURATE |
| 月41h(小)・58h(中)+ 6 年 3 割減 | **CRITICAL 欠落** | **CRITICAL 欠落** |
| 教科担任制 9,400 人定数 | **HIGH 欠落** | **HIGH 欠落** |
| 過労死ライン 月80時間 | **CRITICAL 欠落** | **CRITICAL 欠落** |
| 35人学級 令和7年度 | ✓ ACCURATE | ✓ ACCURATE |
| 教員資格認定試験(情報) | 言及なし | ✓ ACCURATE(令和6再開、p.48) |
| 出典 page 明示の密度 | 末尾箇条書きのみ | 本文中に `(p.NN)` 多用 |
| 幻覚混入(峰岡 / 3000人) | 0 件 | 0 件 |

| 重大度別 ✓ ACCURATE | 5 件 | **7 件(改善)** |

gemma3:12b は qwen2.5:7b に対し CRITICAL 1 件救出 + ACCURATE 2 件追加。一方で月41/58/9400/80 の欠落は両モデル共通で reduce 圧縮ロス由来であり、reduce プロンプトの数値必須リスト指示だけでは解消できないことが確認された。

### 実行コスト

| 指標 | qwen2.5:7b | gemma3:12b |
|---|---|---|
| map 合計 | 1,200.4s | 2,207.8s |
| reduce | 124.2s | 253.2s |
| **合計実時間** | **1,324.6s(22 分)** | **2,460.9s(41 分)** |
| 出力長 | 1,004 chars | 970 chars |

リカバリ込み(qwen2.5:7b はセッション 12 で chunk-5 崩壊 → chunk-5-fix で 5a/5b 再 map 803s 追加)で見ると qwen2.5:7b 2,127.7s vs gemma3:12b 2,460.9s で +15%。gemma3 は chunk 設計済みで 1 回完走。個別 PDF を背景処理する想定なら 41 分は許容範囲。

## 検討した選択肢

### A. gemma3:12b 単独で採用(却下)

条件: 幻覚 0 + CRITICAL 欠落 0-1。

実測: 幻覚 0 ✓ / CRITICAL 欠落 **2 件** ✗ → 不採用。

### B. PoC 中止し別アプローチへ即時切替(却下)

条件: gemma3:12b でも CRITICAL 3 件以上残存。

実測: CRITICAL 2 件(qwen2.5:7b の 3 件から 1 件改善)。改善傾向が確認できており、別アプローチ(クラウド API / 別モデル / RAG 化)への即時切替は早計。

### C. gemma3:12b + post-process 数値 grep retry + 編集者最終監修(採用)

採用案: gemma3:12b を本命とし、以下の補強を組み合わせる。

1. reduce 後の数値 grep 自動チェック(必須数値リストを正規表現でマッチ確認、不足時は当該 chunk 出力を直接参照する 2 段目 reduce)
2. 編集者最終監修(初回 3 件は教育長レベル監修)
3. 対象 PDF 限定(概要版 PDF 20p 程度、または pre-trim 済み本体 PDF)
4. 実行時間 41 分を許容(個別 PDF を背景処理する想定)

→ パターン C を採用。

## 決定

C を採用。具体内容は以下のとおり。

### (1) 採用条件

- **対象 PDF**: 「概要版 PDF 20p 程度」または「pre-trim 済み本体 PDF」のみ。20p を大きく超える本体 PDF は chunk 設計と pre-trim 範囲を都度検討してから着手する
- **使用モデル**: `gemma3:12b`(128K context、Ollama ローカル推論、`qwen2.5:7b` 等への切替は本 ADR を再検討してから)
- **chunk 設計**: pages 単位で chunk あたり約 15,000-18,000 chars 上限(C-1)、末尾の文字化け / 委員名簿等の領域を pre-trim(C-5)
- **post-process**: 必須数値 grep + 不足時 retry を運用前提(C-7/C-8、後続 PR で `fact-check-grep.mjs` を実装)
- **人手 fact-check**: 編集者が原文 PDF と突合して核心数値 / 出典ページ / 幻覚有無の 3 点を確認(C-6)
- **初回 3 件**: 教育長レベル監修(運用安定化までの強化フェーズ。3 件評価後に通常監修体制への移行可否を判断)

### (2) 設計制約 C-1 〜 C-8

PoC セッション 12-15 で観測した実装上の制約を、本 ADR で運用ルールとして固定化する。

- **C-1 chunk size**: 7B/12B + `num_ctx=32768` で chunk あたり 15,000-18,000 chars(約 10-19k prompt tokens)が上限。これを超えると map 段階で context overflow による出力崩壊が発生する
- **C-2 undici headersTimeout**: Ollama への HTTP 呼び出しは `setGlobalDispatcher(new Agent({headersTimeout:0, bodyTimeout:0}))` でタイムアウト無効化必須。デフォルトの 5 分タイムアウトでは 12B モデルの単 chunk map(7-9 分)が完走しない
- **C-3 reduce バイアス**: chunk ごとの抽出品質に偏りがあると reduce 結果も偏る。reduce プロンプトに「全 chunk の論点を均等に統合せよ」のガード文言を含める
- **C-4 末尾文字化け**: PDF 末尾の委員名簿等のグリフ ID 直マッピング文字が context 末端でモデル混乱を誘発する
- **C-5 末尾文字化けは context 上限到達不要で幻覚を呼び込む**: chunk 設計段階で pre-trim が必須(両モデルで有効性確認 ✓、幻覚混入 0 件)
- **C-6 fact-check は人手 grep 突合が必須**: 完全幻覚は出力単体では検出不能(モデルは自信を持って偽情報を出す)、編集者最終監修も必須
- **C-7 数値網羅性**: reduce プロンプトの数値必須リスト指示だけでは不十分。両モデルで月41/58/9400/80 が脱落することを確認。post-process の数値 grep + 不足時 retry が必須。Phase 2 採用判定済(2026-05-17 / ADR 0046、observation-2026-05-17 で本体 100% / 概要版 100% を実証)。2026-05-19 再現性確認(observation-2026-05-19): 文科省通知(中教審第251号踏まえ、tsuuchi.pdf)で通知本文 7/8(87.5%)/ 別添資料2 9/9(100%)/ 別添資料3 9/9(100%)、表面救出率合計 25/26(96.2%)・真の救出率(strict)22/26(84.6%)を実証(obs-17 と通算 2 件目で採用判定固定)
- **C-8 出力長下限**: 73p 答申で reduce 出力 970-1004 chars は不足。プロンプトの「圧縮しない」明示も効きが限定的、出力長下限保証は post-process retry を C-7 と統合運用する

### (3) ADR 0008 §3 への参照

ADR 0008 §3「要約は RSS 配信のものをそのまま使う、独自に長くしない、要約しない、AI で書き換えない」は他媒体配信の RSS `<description>` 改変禁止が原典であり、edu-watch が公的一次ソース PDF の本文から独自に生成する要約はこれに該当しない。両者整合のため ADR 0008 §3 にも本 ADR を参照する例外条項を本 PR で追記する。

## 影響と運用

### 効果

- 中教審答申のような 70p+ 公的 PDF を編集者が digest 化する所要時間を実質短縮(編集者の論点抽出工程に AI 補助を投入)
- 出典ページ明示が密(`(p.NN)` 多用)で、読者の原文照会・編集者の fact-check 効率に貢献
- ローカル推論(Ollama)のためコンテンツ・利用ログが外部 API に流出しない

### 監視 / リスク観測項目

- post-process 数値 grep の誤検知 / 取りこぼし(後続 PR `fact-check-grep.mjs` 実装後に観測開始、必須数値リストの過不足を月次でメンテ)
- 編集者監修工数(初回 3 件で計測。教育長レベル監修コストが運用継続可能か判断、不可なら運用停止して別アプローチへ)
- gemma3:12b 出力の品質変動(モデルバージョンアップ / Ollama アップデートで挙動が変わる可能性、四半期ごとに再評価)
- Ollama / M1 16GB 環境のメモリ圧迫(他作業との競合、PoC 中の background 実行で観測済み)

## 撤回 / 再検討の条件

- 編集者監修コストが運用継続不能水準に達した場合(初回 3 件評価後に判断)
- post-process 数値 grep + retry でも CRITICAL 欠落が常態化した場合(月次品質ログで判定)
- gemma3:12b の挙動が大きく変動し、`fact-check-v3.md` の品質基準(幻覚 0 / CRITICAL 欠落 2 件以下 / ACCURATE 7 件以上)を満たさなくなった場合
- Ollama / ローカル推論環境の維持コストが運用利益を超えた場合(クラウド API 移行 ADR を別途起票)
- 公的一次ソース PDF の利用範囲を超える要件(他媒体本文の AI 要約 / 著者識別を含む処理 等)が必要になった場合(別 ADR で運用方針を再検討)
