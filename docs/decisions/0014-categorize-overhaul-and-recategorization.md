# 0014. categorize ロジックの改善と既存記事の再カテゴライズ

- 状態: 採用(初版 categorize の欠陥を是正)
- 日付: 2026-04-28
- 関連 PR: #18(初版 categorize)、TBD(本 ADR の実装)

## 背景

Sprint 3 のカテゴリ別ページ実装の準備で蓄積記事のカテゴリ分布を確認したところ、161 件中 64 件が「その他」に流れる一方、「いじめ」「国際・海外」などほぼ空のカテゴリが複数ある状態だった。

調査の結果、原因は 2 つ:

1. **mext / chukyo の summary が null**: parser が RSS の `<description>` を取れていないケースがあり、`title + summary` のテキスト判定で title 単体に落ちる
2. **mext / chukyo の title が事務的**: 「配付資料」「審議会(第N回)」「ワーキンググループ」などで、初版 categorize.ts のキーワード網にヒットしない

加えて、初版のキーワード網そのものが薄く、「学力調査」「報告書」「シンポジウム」「教育委員会」「指針」「ガイドライン」「養護教諭」「研修」「生成 AI」「校務 DX」など、教育報道で頻出するキーワードが大量に欠落していた。結果、本来は明確なカテゴリに該当する記事が「その他」に滑り続けていた。

## 検討した選択肢

### A. categorize 改善 + 既存記事の再カテゴライズ(採用)

(1) sourceId デフォルトカテゴリ(mext/chukyo → 政策・制度、nier → 研究・エビデンス)を導入。キーワード未ヒット時のフォールバックとして使う
(2) 各カテゴリのキーワード網を拡充
(3) 既存 161 件を一度きりのマイグレーション(`scripts/recategorize.ts`)で再分類

ADR 0010 との関係: ADR 0010 は「RSS が再配信した記事の `collectedAt` を上書きしない」が趣旨。**categories は外部から取り込んだデータではなく内部処理の結果**(派生データ)であり、ロジック改善に伴う再計算は別概念。`collectedAt` / `publishedAt` / `id` 等は触らない。

### B. categorize 改善のみ、既存は据え置き(却下)

新規取り込みから新ロジックが効くが、既存 161 件のカテゴリが古いままで UI 上の体験が中途半端になる(その他ページに過去の山、各カテゴリは新着のみ)。カテゴリ別ページが実用的になるのに数週間かかる。

### C. カテゴリ別ページを後回し、媒体別ページに進む(却下)

カテゴリ自体の信頼性を放置することになる。Sprint 3 のロードマップでカテゴリ別ページは媒体別と並列の柱で、カテゴリの質を諦めるのは Sprint 全体の価値を下げる。

## 決定

A を採用。

### categorize.ts の改善

- `categorize(input)` の入力型に `sourceId?: string` を追加。`normalize.ts` 側で `categoriesFor({ ...raw, sourceId: parser.sourceId })` を渡すよう更新
- キーワード未ヒット時は `SOURCE_DEFAULTS[sourceId]` を試し、それも無ければ「その他」
- 各カテゴリのキーワード網を拡充(具体的な追加分はソース参照、概要):
  - 政策・制度: 審議会 / WG / 配付資料 / 指針 / ガイドライン / 教育委員会 / 改訂 など
  - 研究・エビデンス: 学力調査 / 報告書 / 統計 / シンポジウム / 公表 など
  - 教員・働き方: 教師 / 教職 / 養成 / 研修 / 養護教諭 / 校長 など
  - ICT / GIGA: 生成 AI / AI / 校務 DX / プログラミング教育 / STEM / STEAM など
  - 不登校: 別室登校 / 教育機会確保法 / フリースクール
  - 国際・海外: TALIS / ユネスコ / UNESCO

### scripts/recategorize.ts(一度きり)

`src/data/articles/*.json` を全て読み込み、各 Article の `categories` を新ロジックで再計算して書き戻す。`collectedAt` / `publishedAt` / `id` 等は不変。`--dry-run` で件数だけ確認できる。

## 影響

実測の前後比較(全 25 ファイル / 161 記事 / カテゴリ重複含む全付与数 182):

| カテゴリ | before | after |
|---|---|---|
| その他 | 64 | **28**(半減) |
| 研究・エビデンス | 55 | 65 |
| 政策・制度 | 27 | **52**(約 2 倍) |
| 教員・働き方 | 25 | 39 |
| ICT / GIGA | 5 | **17**(3 倍超) |
| 不登校 | 4 | 4 |
| 国際・海外 | 2 | 2 |
| いじめ | 0 | 0 |

52 記事の categories が更新された(全体の 32.3%)。`collectedAt` / `publishedAt` / `id` は無変更。「いじめ」「国際・海外」が低件数なのはルール側ではなく、収集期間の話題が偏っているため(キーワードを増やしても false positive を増やすだけ)。

## 撤回 / 再検討の条件

- カテゴリ自動判定を LLM ベースに置き換える場合 → 本 ADR の RULES と SOURCE_DEFAULTS は破棄、`categorize(input)` の signature を維持して内部実装だけ差し替え
- false positive(無関係記事が大量に拾われる)が観察された場合 → ルールの個別キーワードを削る or `categorize` の前処理として title/summary の正規化(全角/半角・記号類)を強化
- カテゴリ自体の構成(8 カテゴリ)を変える場合 → 本 ADR は破棄し新規 ADR を起こす
