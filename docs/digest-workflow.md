# 週次ダイジェスト 共同制作フロー

EduWatch の週次ダイジェスト（`src/content/digests/YYYY-MM-DD.md`）は、**編集者主導の共同制作**で作る。**論点・主張・本文は編集者が担い、Claude は事実調査と清書を担う。** Claude が論点や本文を先回りして書かない。

> **背景**: digest #5（2026-05-23 号）で Claude が単独で全文を執筆した結果、論点・本文が編集者の意図と乖離し、PR #183 を撤回・再構築した。その再発防止として本フローを定める。

## 役割分担

| 担当 | 内容 |
|------|------|
| **編集者** | 採用記事の確定／節構成・順序の決定／各節の**論点（本文を含む）の執筆**／PR レビュー・マージ |
| **Claude** | 候補の洗い出しと**事実の調査・提示**（一次ソースで裏取り）／編集者の論点・本文の**清書**／ファイル作成・PR 起票 |

## 手順

### 全体

1. **採用記事の確定**（編集者）。Claude は候補一覧と事実確認を補助する。
2. **節構成・順序の決定**（編集者）。複数記事を 1 セクションに束ねる判断もここで行う。

### 各セクションごとに繰り返す

1. **Claude: 事実を提示** — 中立・出典つき。数値・固有名詞は一次ソースで裏取りし、推測で書かない。裏取りは生記事テキスト（`curl`）と一次 PDF（`pdftotext`）の逐語照合で行い、WebFetch の要約には依存しない（要約は非決定的で内容が揺れるため）。
2. **編集者: 論点（本文を含む）を提示** — 編集判断・主張・本文ドラフトは編集者が書く。
3. **Claude: 清書して提示** — 編集者の論点・本文を digest 文体に整える。**論点の中身は変えない。** 行うのは次のみ。
   - 事実・数値・固有名詞の補強と、**記事化されていない補足資料**（統計・自治体ページ・報告書 PDF など）への inline link 付与。**記事カード（Referenced articles）で読めるニュースの出典（新聞記事・その一次ソース）は本文にリンクを置かない**
   - 関連する EduEvidence 戦略・コラム、過去 digest への横断リンク
   - 「私」を主語にしない観察フレーミングへの統一
   - 太字・体裁の調整
   - 清書の最後に [deslop-checklist.md](./deslop-checklist.md) を通し、定型的・機械的な言い回し（擬人化主語・二項対比の連打・まとめの定型 等）を除く。**論点と数値は対象外**
4. **編集者が確認** → 次の節へ。

### 最後

1. 全節の清書が編集者の確認を得たら、**後段検証ゲート**（下記「後段検証ゲート（最終本文の逐語照合）」節）を回す。最終本文を一次資料と claim 単位で逐語照合し、GO（機微ティアは編集者 sign-off）を得るまで PR に進まない。
2. ゲート通過後、**Plan Mode** で対象ファイルと出典を宣言してから作成する。
3. `npm run check` / `npm run build` と生成 HTML の確認を通す。
4. 同 PR 内で `changelog.astro` にユーザー価値のある変更を 1 行追記する（本文の読者体験が変わる場合）。
5. ブランチを切って PR を起票する。**マージは編集者が行う**（Claude は自動マージしない）。

## 後段検証ゲート（最終本文の逐語照合）

事実検証には**清書前（pre-draft）**と**清書後（post-draft）**の 2 つの関門がある。

- **pre-draft**: 各セクション手順 1 と `digest-fact-checker`（執筆前ゲート）。編集者が「こう書きたい」と示した主張を、本文を起こす前に一次資料で確認する。
- **post-draft**: 本節。**公開されるそのままの最終本文**を、一次資料と claim 単位で逐語照合する。

> **なぜ post-draft が要るか**: 2026-06-27 の W26 号で、`digest-fact-checker` の「生徒**ら**2人が死亡」（＝生徒とその他で計 2 人・内訳未確定）という pre-draft の表現が、清書の過程で「生徒2人」へ**過度に特定**された。実際は生徒 1 人＋船長 1 人で、死者の身元を誤った。pre-draft 検証は通っていたが、**清書で混入した誤りを最終本文で照合する関門が無かった**ため、公開直前に編集者が気づくまで残った（PR #330。教訓は memory `feedback_ambiguous_collective_count`）。ゲートの本質は「清書後の本文を、清書の経緯を信用せず原典から引き直す」ことにある。

### 何を・どう回すか

1. 対象は**最終本文（公開されるそのままの文）の全 claim**＝数値・固有名詞・日付・引用・死傷者・法令・金額。
2. `digest-fact-checker` を**後段（post-draft）モード**で回す。入力は清書済みの最終本文と原典のみ。清書の経緯や「編集者がこう言ったから」は根拠にしない。
3. **独立第 2 パス**を併走させる（`edu-adversarial-verifier` と同じ独立反証の原則）。最終本文を「正しいと仮定せず」、原典だけからゼロに引き直し、清書の言い回しとの食い違いを能動的に探す。研究の効果量・出典帰属を含む回（例: 文科省委託報告書の数値）は `edu-adversarial-verifier` 自体を併用する。
4. 判定が STOP / HOLD の間は **PR に進まない**。修正 → 再照合してから先へ進む。

### 抽出表（過度特定・改変の機械的検出）

最終本文から機微・要検証の要素を 1 項目 1 行で抜き出し、**原典の語を要約せず一字一句コピー**して照合する。本文の言い換えではなく原典のコピーを並べることで、清書での詰め過ぎ・正規化・改変が表で可視化される。

| 本文の記述 | 原典の逐語（コピー） | 出典 URL | 一致 |
|---|---|---|---|
| （例）生徒2人が死亡 | 「生徒**ら**2人が…」／ 文科省 PDF「生徒が死亡」 | （各 URL） | ✗ 過度特定 |

### 機微事実ティア（最重要・load-bearing）

**死傷者（死者・負傷者の数／身元／年齢）・法令（成立・公布・施行・条項・罰則）・金額（予算・賠償・コスト）**は、誤れば訂正が要る最も機微な事実。通常の逐語照合に加え、次の 4 点を**すべて**満たすまで公開しない。

1. **2 つの独立した一次源で照合する**。単一報道・単一の二次まとめでは不可（例: 所管庁の一次 PDF ＋ 複数の独立報道、または当事者発表＋公的記録）。
2. **逐語**。原典の表現を一字一句で確認し、要約・正規化・概数化しない。
3. **人物単位の内訳照合**。「N 人」を集団のまま扱わず、一人ひとりの属性（生徒／船長／教員／年齢）に分解し、各人が一次で確認できるまで人数・身元を断定しない。
4. **編集者の sign-off**。機微ティアに該当する記述は、最終本文の該当文を編集者が個別に承認する。

### 「〜ら N 人」曖昧フラグ

「〜ら N 人」「〜など N 件」「関係者 N 名」のような**集合表現**（X とその他で計 N の含意）を最終本文・一次資料に見つけたら、内訳が一次で確定するまで次のいずれかにする。

- 「ら」「など」などの**曖昧さを保持する**（"生徒ら2人" を "生徒2人" に詰めない）。
- **人数・身元の断定を避ける**。

過度な特定（over-specification）は、根拠の無い特定を足す点で捏造と同じ訂正リスクを生む。

### 適用範囲

- post-draft の**逐語照合＋抽出表は毎回（全 digest）**実施する。軽量で、数値・固有名詞・日付の取り違えを広く拾う。
- **機微事実ティアの 4 点規律は、claim が死傷者・法令・金額に該当する回のみ**発火する。重い手順をリスクに比例させ、毎週の運用が形骸化しないようにする。

## Claude がやってはいけないこと

- 論点・本文を先回りして書く（手順 2 は編集者の領域）。
- 清書時に編集者の論点の中身を変える・足す（手順 3 は事実補強と体裁のみ）。
- 全文を単独で執筆する。
- 編集者の確認を飛ばして次の節やファイル作成に進む。

## 執筆上の約束（要点）

- **文体**: digest 本文は敬体（です・ます調）で統一する。
- **事実段落の必須要素と簡潔さ**: いつ／誰が／何を・どうした＋見出しの数値 1〜2 個に絞る。内訳・前回比・手続きの詳細・機関や通知の正式名称は本文に詰め込まず、リンク先に委ねる（読者は必要なら出典に飛ぶ）。論点段落が参照する数値だけは事実側に残す。事実段落は 1〜2 文を目安に、最初の清書から簡潔に書く。出典はセクションの記事カード（Referenced articles）に委ね、本文中に出典リンクは置かない。記事化されていない補足資料のみ本文に inline link を付ける。
- **セクション統合**: 同テーマ／論点交差／週内同時性／編集者軸の意義の 4 軸のうち 2 軸以上で統合を検討。1 セクション最大 3 記事。
- **段落数の目安**: 単独 1 記事 = 3-4／統合 2 記事 = 4-5／統合 3 記事 = 5-6／digest 全体 = 12-18。上限は原則超えない。
- **heading**: 「論点 — 対象」のダッシュ区切り（`—` U+2014）、25-45 文字目安。断定指示（「〜すべき」）や一方的批判は避け、観察フレーミング（「残る／移る／見えてくる」）や問いかけで立てる。
