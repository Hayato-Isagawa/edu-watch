# 週次ダイジェスト 共同制作フロー

EduWatch の週次ダイジェスト（`src/content/digests/YYYY-MM-DD.md`）は、**編集者主導の共同制作**で作る。**論点・主張・本文は編集者が担い、Claude は事実調査と清書を担う。** Claude が論点や本文を先回りして書かない。

> **背景**: digest #5（2026-05-23 号）で Claude が単独で全文を執筆した結果、論点・本文が編集者の意図と乖離し、PR #183 を撤回・再構築した。その再発防止として本フローを定める。

## 役割分担

| 担当 | 内容 |
|------|------|
| **編集者** | 採用記事の確定／節構成・順序の決定／各節の**論点（本文を含む）の執筆**／PR レビュー・マージ |
| **Claude** | 候補の洗い出しと**事実の調査・提示**（一次ソースで裏取り）／編集者の論点・本文の**清書**／ファイル作成・PR 起票 |

## 手順

### 全体

1. **採用記事の確定**（編集者）。Claude は候補一覧と事実確認を補助する。
2. **節構成・順序の決定**（編集者）。複数記事を 1 セクションに束ねる判断もここで行う。

### 各セクションごとに繰り返す

1. **Claude: 事実を提示** — 中立・出典つき。数値・固有名詞は一次ソースで裏取りし、推測で書かない。
2. **編集者: 論点（本文を含む）を提示** — 編集判断・主張・本文ドラフトは編集者が書く。
3. **Claude: 清書して提示** — 編集者の論点・本文を digest 文体に整える。**論点の中身は変えない。** 行うのは次のみ。
   - 事実・数値・固有名詞の補強と、一次ソースへの inline link 付与
   - 関連する EduEvidence 戦略・コラム、過去 digest への横断リンク
   - 「私」を主語にしない観察フレーミングへの統一
   - 太字・体裁の調整
4. **編集者が確認** → 次の節へ。

### 最後

1. 全節合意後、**Plan Mode** で対象ファイルと出典を宣言してから作成する。
2. `npm run check` / `npm run build` と生成 HTML の確認を通す。
3. 同 PR 内で `changelog.astro` にユーザー価値のある変更を 1 行追記する。
4. ブランチを切って PR を起票する。**マージは編集者が行う**（Claude は自動マージしない）。

## Claude がやってはいけないこと

- 論点・本文を先回りして書く（手順 2 は編集者の領域）。
- 清書時に編集者の論点の中身を変える・足す（手順 3 は事実補強と体裁のみ）。
- 全文を単独で執筆する。
- 編集者の確認を飛ばして次の節やファイル作成に進む。

## 執筆上の約束（要点）

- **事実段落の必須要素**: いつ／誰が／何を・どうした／一次出典 URL。出典は本文中の inline link で統一する。
- **セクション統合**: 同テーマ／論点交差／週内同時性／編集者軸の意義の 4 軸のうち 2 軸以上で統合を検討。1 セクション最大 3 記事。
- **段落数の目安**: 単独 1 記事 = 3-4／統合 2 記事 = 4-5／統合 3 記事 = 5-6／digest 全体 = 12-18。上限は原則超えない。
- **heading**: 「論点 — 対象」のダッシュ区切り（`—` U+2014）、25-45 文字目安。断定指示（「〜すべき」）や一方的批判は避け、観察フレーミング（「残る／移る／見えてくる」）や問いかけで立てる。
