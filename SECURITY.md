# Security Policy

## 脆弱性の報告

EduWatch JP のコード・インフラに関する **脆弱性** を見つけた場合は、公開の Issue ではなく、以下のメールにご連絡ください。

- **notify@edu-evidence.org**

メールには以下を含めていただけると助かります:

- 影響範囲(リポジトリ / 本番サイト / 依存ライブラリ等)
- 再現手順(可能な限り具体的に)
- 想定される影響度(情報漏えい / 任意コード実行 / 可用性 など)

受領後、72 時間以内に初回返信します(平日基準)。調査・修正のために詳細の追加質問をお願いする場合があります。

## 報告してほしい例

- 本サイトの **XSS / オープンリダイレクト**
- CSP ヘッダーや Email 経路の設定不備
- ビルド成果物(`dist/`)に混入している秘匿情報
- RSS / 外部リソース経由の **prototype pollution や SSRF**

## 報告不要な例

- 使っていないブラウザ古バージョンでのみ発生する表示崩れ
- 記事内容に関する事実誤認(コンテンツの誤りは Issue テンプレート「コンテンツの誤り報告」をご利用ください)

## 開示方針

修正がリリースされるまで、報告者には公開を控えていただけると助かります。修正リリース後、CHANGELOG または該当 PR で謝辞(希望時のみ)を記載します。
